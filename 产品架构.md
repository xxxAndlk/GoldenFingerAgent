# 金手指 Agent 系统 — 产品架构

> **定位**：本地优先、持续进化的个人 Agent 系统
> **状态**：v0.1.0 — MVP 已实现，五域流水线全部跑通
> **最后更新**：2026-05

---

## 一、产品定义

金手指是一个**本地优先、持续进化**的个人 AI Agent 系统。它接收用户的任意问题，自行拆解、规划、执行、校验，并将每一次经验沉淀为可复用的「功法」——越用越强。

### 核心隐喻

| 网文概念 | 产品映射 | 实现模块 |
|----------|----------|----------|
| 天机推演 | 分析规划 | `domain_analysis.py` |
| 施法执行 | 执行调度 | `domain_execution.py` |
| 验道校验 | 结果校验 | `domain_verification.py` |
| 刻碑沉淀 | 持久进化 | `domain_persistence.py` |
| 内外结界 | 隔离安全 | `domain_isolation.py` |
| 宿主 | 用户 | `HostProfile` (画像) |
| 灵根 | 认知风格 | `SpiritRoot` (五行属性) |
| 功法/Skill | 可复用技能 | `skills/` 目录 |

## 二、五域架构（当前实现）

```
用户问题
    │
    ▼
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│① 天机推演 │──▶│② 施法执行 │──▶│③ 验道校验 │──▶│④ 刻碑沉淀 │
│ 分析规划  │   │ 执行调度  │   │ 结果校验  │   │ 持久进化  │
└──────────┘   └──────────┘   └──────────┘   └────┬─────┘
     │                                             │
     └──────────── ⑤ 内外结界（贯穿全程） ◀──────────┘
```

### 域一：天机推演（分析规划域）

**职责**：将用户自然语言问题拆解为原子任务 DAG

**流程**：
1. **意图分类** — 评估复杂度（simple_qa / skill_single / skill_chain / long_running）
2. **任务拆解** — LLM 驱动的 DAG 生成，包含依赖关系
3. **Skill 匹配** — 关键词匹配已有 Skill（知识汲取术/代码辅助术/文件操作术）
4. **提示词组装** — 注入宿主画像上下文 + Skill 知识 + 安全约束
5. **输出 TaskPlan** — 包含拓扑分层的原子任务列表

**落地模块**：`domain_analysis.py`（`PlanGenerator` 类，约 340 行）

### 域二：施法执行（执行调度域）

**职责**：按 TaskPlan 拓扑序执行任务，管理 LLM ↔ Tool 交互循环

**流程**：
1. **拓扑分层调度** — 按 DAG 层级调度，同层任务可并行
2. **单任务执行循环** — LLM 调用 → 解析 Tool Call → 安全检查 → 工具执行（最多 5 轮）
3. **安全防御** — 提示词注入检测 + 权限校验 + 参数净化 + 结果校验
4. **输出 ExecutionReport** — 包含工具调用日志和异常记录

**落地模块**：`domain_execution.py`（`ExecutionOrchestrator` 类）

### 域三：验道校验（结果校验域）

**职责**：三层校验模型确保执行结果质量

**流程**：
1. **结构校验** — 检查输出格式、完整性、工具调用是否异常
2. **内容校验** — LLM-as-Judge 评估相关性、完整性、准确性
3. **步骤回放** — 验证关键工具操作是否生效
4. **失败处理** — 重试 → 回退 → 重构 → 请求用户澄清

**落地模块**：`domain_verification.py`（`VerificationEngine` 类）

### 域四：刻碑沉淀（持久进化域）

**职责**：将执行经验转化为可复用的 Skill 知识

**流程**：
1. **执行摘要生成** — LLM 生成结构化摘要（做了什么→怎么做的→遇到什么问题→怎么解决）
2. **Skill 缺口分析** — 检测未被覆盖的任务步骤
3. **Skill 更新/新建** — 追加知识条目 + 更新向量库 + 调整统计
4. **宿主画像更新** — 更新灵根属性 + 累计修炼时间 + 检测境界突破

**落地模块**：`domain_persistence.py`（`PersistenceEngine` 类）

### 域五：内外结界（隔离安全域）

**职责**：贯穿全流程的横向安全切面

**数据三级分级**：
| 等级 | 名称 | 范围 | 出境规则 |
|------|------|------|----------|
| Level 0 | 紫府级 | 真实姓名/坐标/收入 | 禁止出境 |
| Level 1 | 灵台级 | 能力坐标/学习偏好 | 脱敏后内部使用 |
| Level 2 | 识海级 | 境界等级/通用标签 | 可出境 |

**关键能力**：
- 出境脱敏（PII 检测 + 坐标模糊化）
- 入境过滤（XSS/注入/路径穿越检测）
- 画像演化（从交互中提取更新信号）

**落地模块**：`domain_isolation.py`（`EgressAnonymizer`、`IngressFilter` 类）

## 三、技术栈（已实现）

```
┌─────────────────────────────────────────────────────┐
│              交互界面层（3 种模式）                     │

│  └──────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────┤
│              FastAPI 后端 (api.py)                    │
│  REST API + SSE 流式推送                             │
├─────────────────────────────────────────────────────┤
│           Harness 核心调度 (harness.py)                │
│  ┌──────────┬──────────┬──────────┬──────────┐      │
│  │ 天机推演  │ 施法执行  │ 验道校验  │ 刻碑沉淀  │      │
│  └──────────┴──────────┴──────────┴──────────┘      │

```

| 层级 | 选型 | 用途 |
|------|------|------|
| LLM 调用 | `httpx` 直调 OpenAI / Anthropic API | 模型调用、流式输出 |
| Web 后端 | `FastAPI` + `uvicorn` + SSE | REST API + 流式事件推送 |
| Web 前端 | `Vue 3` + `TypeScript` + `Vite` + `marked` | 浏览器可视化界面 |
| 终端 TUI | `Textual` + `Rich` | 命令行可视化交互 |
| 数据模型 | `pydantic` v2 | 全量数据模型（`models.py`，333 行） |
| 向量存储 | `ChromaDB`（嵌入式） | Skill 知识向量检索 |
| 关系存储 | `SQLite3`（Python 标准库） | 宿主画像/Skill 元数据/执行日志 |
| 配置管理 | `python-dotenv` + 环境变量 | API Key / 目录 / 超时配置 |
| 异步 | `asyncio` | 全异步架构 |
| 模板引擎 | `Jinja2` | Skill 提示词模板 |

## 五、内置 Skill 与工具

### Skill（3 个）

| Skill | 名称 | 适用场景 | 所需工具 |
|-------|------|----------|----------|

| 工具 | 说明 | 安全限制 |

| `shell_exec` | 执行 Shell 命令 | 危险命令检测，60s 超时 |

## 八、安全模型

### 出境安全（用户数据 → LLM API）
- 出境前经 `EgressAnonymizer` 脱敏处理
- PII 检测：手机号/身份证/邮箱 → 替换为占位符
- 坐标模糊化
- 仅传递识海级数据（境界、通用标签）

### 入境安全（LLM 响应 → 用户环境）
- `IngressFilter` 过滤外部返回内容
- XSS / SQL 注入 / 命令注入 / 路径穿越检测
- 模板注入检测

### 执行沙箱
- 文件操作：仅允许工作目录、data 目录、系统临时目录
- Shell 命令：正则检测 10+ 种危险模式
- 超时控制：工具执行 60s，LLM 调用 120s

## 九、当前实现 vs v2.0 远景规划

| 维度 | 当前实现 (v0.1.0) | v2.0 远景 |
|------|-------------------|-----------|
| 执行编排 | asyncio 协程调度 | Temporal.io 工作流引擎 |
| 沙箱隔离 | 路径限制 + 正则检测 | Docker + gVisor 内核隔离 |
| 权限控制 | 硬编码规则 | OPA 声明式策略 |
| 注入检测 | 正则 + LLM-as-Judge | LLM Guard 专用库 |
| 宿主画像加密 | SQLite 明文 | SQLCipher 透明加密 |
| PII 脱敏 | 自定义正则规则 | Microsoft Presidio |
| 前端界面 | Vue 3 基础版 | 完整 Web UI（多宿主、移动端） |
| Skill 生态 | 3 个内置 Skill | 开放 Skill 市场 |
| 多宿主协同 | 无 | 宗门/社群系统 |
| 因果推演 | 无 | 长期规划与预测 |

详细远景规划见 `金手指Agent产品架构v2.md`。

## 十、开发指南

### 启动开发环境

```bash
# 后端
cd GoldenFinger
pip install -r requirements.txt
pip install -e .
cp .env.example .env   # 编辑填入 API Key

# 前端
cd frontend
npm install
npm run build           # 构建后 gfa --web 自动加载
```

### 运行

```bash
gfa                    # 终端 TUI（默认）
gfa --web              # Web 界面（自动打开浏览器）
gfa --status           # 查看宿主状态
gfa "帮我写一个脚本"    # 单次命令
```

### 技术约束

- Python 3.10+
- 需要至少配置一个 LLM API Key（OpenAI 或 Anthropic）
- 前端需要 Node.js 18+ 构建
